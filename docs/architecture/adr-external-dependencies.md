---

## 1. Formily（动态表单）

| 项 | 决策 |
|:---|:---|
| **用途** | 工单类型动态字段、低代码表单设计器（PRD §3.3.1） |
| **MVP** | **S2 前引入**；S0/S1 不阻塞管理端骨架 |
| **方案** | AdminWeb 嵌入 Formily 2.x；表单 schema JSON 存 `custom_field_config` |
| **降级** | S2 初版可用**静态字段**工单类型，Formily 作为 S2 中后 Story |
| **验证** | S2 启动前做 1 页 POC（创建/渲染最小表单） |

---

## 2. MinIO（S3 协议对象存储）

| 项 | 决策 |
|:---|:---|
| **用途** | 工单/咨询附件 |
| **环境** | 已部署 `127.0.0.1:30090`（API）、`:30091`（Console） |
| **MVP** | **S2 附件 Story** 接入 SDK；S0/S1 可占位不上传 |
| **降级** | 本地磁盘临时存储（仅 dev，不用于生产） |
| **配置** | 凭据不入库；Spring 配置 + 环境变量 |

---

## 3. SMTP（邮件通知）

| 项 | 决策 |
|:---|:---|
| **用途** | 工单/SLA 邮件通知 |
| **MVP** | **默认仅站内信**；SMTP 可选，不阻塞北极星闭环 |
| **降级** | 未配置 SMTP 时跳过邮件发送，不报错阻断主流程 |
| **接入** | S3 SLA/通知 Epic 再评估是否必开 |

---

## 4. Redis

| 项 | 决策 |
