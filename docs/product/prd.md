## 1. 产品概要

### 1.1 产品定位
UnionDesk 是一个企业级、多业务域、可配置的客户服务中台。它帮助企业统一管理**工单、在线咨询、反馈建议**三类服务请求，通过"业务域"实现不同品牌/业务线的独立运营与权限隔离。

### 1.2 核心价值
- **多业务域完全隔离**：一套系统，管理多个独立品牌，互不干扰。
- **统一身份，两端分离**：一个账号主体，可同时是A业务的客户、B业务的客服。
- **动态工单表单**：各业务域可自定义工单字段和状态流程。
- **完整服务闭环**：在线咨询 → 转工单 → SLA保障 → 满意度评价。

### 1.3 用户角色速览

| 角色 | 核心职责 | 工作端 |
| :--- | :--- | :--- |
| **客户** | 提交工单、在线咨询、查看进度、评价服务 | 客户端 |
| **客服** | 处理工单、接待咨询、使用快捷回复 | 员工端 |
| **域管理员** | 配置业务域（工单类型/SLA/角色/人员） | 域管理后台 |
| **平台管理员** | 创建业务域、管理全局员工、系统安全设置 | 平台管理后台 |

### 1.4 MVP范围
**本期交付**：工单系统（含SLA）+ 在线咨询 + 反馈/建议 + 多业务域权限隔离。
**本期不做**：知识库、帮助中心、AI回复、高级BI报表。

### 1.5 项目边界

| 范围 | 内容 |
| :--- | :--- |
| **MVP 必做** | 工单系统（含SLA）、在线咨询、反馈/建议、多业务域权限隔离 |
| **明确不做** | 知识库/帮助中心、AI自动回复、可视化BI报表、外部供应商协作门户 |
| **下期规划** | 知识库全流程、Elasticsearch全文搜索、VitePress帮助中心 |

## 2. 全局设计决策

> 本章定义了项目不可动摇的架构基石。所有功能开发必须遵守此处的约定。

### 2.1 统一身份模型
- **一个身份主体**：`identity_subject` 是系统中自然人的唯一标识。一个手机号对应一个主体。
- **两种账号**：一个主体下，可同时拥有 `staff_account`（员工账号）和 `customer_account`（客户账号）。
- **登录与切换**：客户端和员工端分别登录，底层共享认证会话。若同一主体已登录员工端，打开客户端时可免密切换。
- **客户入域**：客户账号需加入某业务域后才能访问该域服务。入域方式由业务域的注册策略控制。

### 2.2 多业务域隔离
业务域是资源隔离的基本单元。每个业务域拥有独立的工单、咨询、人员、配置。

| 策略类型 | 选项 | 说明 |
| :--- | :--- | :--- |
| **可见策略** | `公开` / `仅域内客户可见` / `仅渠道可见` | 控制谁可以发现这个域 |
| **注册策略** | `开放注册` / `仅邀请注册` / `仅管理员添加` | 控制客户如何加入这个域 |

### 2.3 权限体系
平台采用 **"平台角色 + 业务域角色"双层权限**。

- **平台级**：`platform_admin`（平台管理员）拥有创建业务域、管理全局员工的最高权限。
- **业务域级**：每个业务域内置三个角色。
    - `super_admin`：域最高权限，管理域内一切。
    - `domain_admin`：管理域内常规配置与人员。
    - `agent`：基础客服，处理工单、接待咨询。
    - *域管理员可基于"权限项"组合创建最多20个自定义角色。*
- **权限项**：精确到按钮的功能权限，格式为 `模块:操作`，如 `ticket:delete`。数据范围通过 `:self` / `:domain_all` 后缀控制。

### 2.4 全局安全策略
- **认证**：JWT + Refresh Token轮换，支持单账号多会话踢除。
- **安全屏蔽**：全平台统一敏感词库，命中即拦截并告警。
- **审计**：所有管理操作、状态变更、权限修改均写入 `audit_log`，不可删除。

## 3. 功能需求

### 3.1 客户端（用户视角）
*客户端是用户寻求帮助的统一入口。*

**3.1.1 服务请求**
- **提交工单**：用户选择工单类型（问题/反馈/建议），填写动态表单，提交后获得工单号。
- **在线咨询**：用户进入咨询窗口，先排队，接入客服后可实时聊天。可发送文字、图片、附件。
- **反馈/建议**：独立入口，提交对产品/服务的意见。用户端仅展示"已收到/处理中/已完成"状态。

**3.1.2 服务跟踪**
- **我的工单**：查看提交过的所有工单，查看客服公开回复，补充信息，确认解决后评价。
- **咨询历史**：回看历史聊天记录和关联工单。
- **满意度评价**：工单关闭或咨询结束后，用户可在通知中心找到评价入口，进行星级+文字评价。

### 3.2 员工端（客服视角）
*客服在此处理分配给自己的工作。*

**3.2.1 工单处理中心**
- **工单队列**：按域、状态、优先级、SLA状态筛选工单。支持批量领取、批量关闭。
- **工单详情**：左侧时间线（公开回复、内部备注、系统操作），右侧属性卡（状态、优先级、SLA倒计时）。
- **处理动作**：公开回复、内部备注、转派他人、变更状态、请求客户补充信息。
- **SLA感知**：列表页和详情页高亮显示SLA即将超时或已超时的工单。

**3.2.2 在线咨询工作台**
- **接待**：客服手动从队列中接入等待的客户。
- **聊天**：实时收发消息，支持撤回2分钟内的消息。
- **转工单**：一键将当前会话转为工单，自动带入客户信息和会话摘要。

### 3.3 域管理后台（管理员视角）
*域管理员在此配置本业务域的规则与人员。*

**3.3.1 域配置**
- **工单类型设计**：使用低代码表单设计器创建/编辑工单类型，配置其动态字段和状态流程。
- **SLA规则**：配置本域默认SLA和按工单类型的特殊SLA（首响/解决时限、违约动作）。
- **通知模板**：编辑各类事件（工单创建、SLA预警等）的站内信和邮件模板。

**3.3.2 人员与权限**
- **成员管理**：从平台员工库中添加/移除本域成员，并分配角色。
- **客户管理**：查看本域客户列表，手动激活/禁用，打标签。
- **角色管理**：创建自定义角色，勾选所需的权限项组合。

### 3.4 平台管理后台（系统视角）
*平台管理员在此进行跨域的全局治理。*

**3.4.1 业务域管理**
- **创建/管理**：新建业务域（从模板或空白），设置其可见与注册策略，启用/禁用。
- **模板中心**：将某个域的完整配置提炼为模板，供快速创建新域。

**3.4.2 员工与权限管理**
- **员工账号**：全局员工列表，手动创建或CSV批量导入。分配平台角色，或将其加入指定业务域并赋予域角色。
- **离职池**：员工账号删除后进入离职池，其名下的未关闭工单自动退回队列。

**3.4.3 系统设置与安全**
- **全局安全策略**：密码强度、登录失败锁定、IP白名单。
- **安全告警中心**：查看和处理所有派生的安全告警（如暴力破解、高频操作）。

## 4. 非功能需求与验收标准

### 4.1 核心验收标准 (摘录)
- **核心闭环**：至少3个业务域下，客户提单 → 客服处理 → 客户查单的流程可闭环。
- **权限隔离**：A域客服不能看到B域工单；自定义角色只能访问被授权的菜单和按钮。
- **安全验证**：关键操作（删除域、全量导出）需密码二次验证，并写入审计日志。
- **SLA验证**：可配置工作日历，SLA计时准确，临近/超时预警正常触发。
- **环境交付**：提供一键`docker-compose.yml`，可跑通全部P0验收用例。

### 4.2 非功能要求
- **性能**：工单列表常用查询 P95 < 1s。
- **安全**：全站HTTPS，密码Argon2哈希，JWT认证。
- **终端**：客户端/员工端需响应式适配手机、平板、桌面端。
- **浏览器**：支持Chrome/Edge/Firefox/Safari最近两个主版本。
