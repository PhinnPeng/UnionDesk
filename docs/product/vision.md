## 1. 一句话定位

UnionDesk 是企业级、**多业务域隔离**的客户服务中台，统一管理工单、在线咨询与反馈建议，支撑不同品牌/业务线独立运营。

## 2. 北极星指标


| 类型      | 指标                         | 说明                                        |
| ------- | -------------------------- | ----------------------------------------- |
| **主指标** | **MVP 内 1 个业务域工单闭环率 100%** | 至少 1 个已启用域完成：客户提单 → 客服处理 → 客户可查进度与回复 |
| **辅指标** | 跨域数据越权事故 **0 次**           | 含 API 与界面；以 P0 验收用例为准                     |
| **辅指标** | P0 核心验收用例通过率 **100%**      | 详见后续 `docs/qa/acceptance-test-catalog.md` |


## 3. MVP 成功定义

同时满足以下条件，视为 MVP 交付成功：

1. **域内闭环**：至少 **1 个**已启用业务域下，工单最小闭环可演示（产品仍支持多域；北极星以 1 域为准）。
2. **权限隔离**：A 域角色无法访问 B 域工单/咨询数据；自定义角色仅见授权菜单与按钮。
3. **安全与审计**：删除业务域、全量导出等高危操作需二次验证并记入审计。
4. **SLA（v1）**：可配置基础 SLA 规则，列表/详情可感知临近或超时（完整工作日历引擎按 Backlog 分期）。
5. **环境交付**：`docker-compose.yml` 一键启动后，可跑通上述核心路径。

## 4. 非目标（Out of Scope）

本期**明确不做**：

- 知识库 / 帮助中心 / VitePress 文档站
- AI 自动回复 / 智能客服
- 高级可视化 BI 报表
- 外部供应商协作门户
- **国际化（i18n）**
- 历史工单 / 客户数据批量迁移导入

## 5. 目标用户


| 角色        | 诉求摘要               |
| --------- | ------------------ |
| **客户**    | 提单、咨询、查进度、评价       |
| **客服**    | 处理工单、接待咨询          |
| **域管理员**  | 配置本域工单类型、SLA、角色与人员 |
| **平台管理员** | 创建业务域、全局员工与安全治理    |
