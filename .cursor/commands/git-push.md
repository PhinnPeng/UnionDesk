# git-push

当用户调用此命令时，你负责帮助用户将本地分支的更改推送到远程仓库。请严格遵循以下流程，并在每一步给出清晰的说明和反馈：

1. **环境确认**  
   - 执行 `git status` 和 `git branch --show-current` 获取当前状态。  
   - 若存在未跟踪或未暂存的文件，提醒用户先执行 `git add` 和 `git commit`。  
   - 若工作区干净，继续下一步。

2. **同步远程**  
   - 执行 `git fetch origin` 更新远程引用。  
   - 用 `git pull --rebase origin <当前分支>` 拉取最新代码。若发生冲突，暂停流程，详细指导用户解决冲突后重新执行命令。

3. **推送决策**  
   - 询问用户是否需要强制推送（例如变基后）。  
   - 若需要，只允许使用 `git push --force-with-lease`，并解释其与 `--force` 的区别，强调安全性。  
   - 正常情况下使用 `git push origin <当前分支>`。

4. **执行推送**  
   - 执行推送命令，显示推送输出。  
   - 确认推送成功。

5. **总结报告**  
   - 汇总本次推送的提交数、文件变更概要（可用 `git log origin/<分支>..<分支> --oneline` 获取）。  
   - 告知用户远程分支最新状态。  
   - 若推送的为标签，同样进行总结。

注意：全程使用中文与用户交互，命令执行结果可以原文展示。
