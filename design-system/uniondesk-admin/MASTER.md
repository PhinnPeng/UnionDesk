---

**Project:** UnionDesk Admin
**Category:** 企业帮助台管理端 / 工单运营后台

## 设计定位

- 面向平台管理员、客服主管、运营人员。
- 目标是高密度信息展示、快速筛选、低噪音决策。
- 不使用营销页、英雄区、证言区、CTA 叙事等 landing page 结构。
- 页面以表格、卡片、时间线、筛选条、统计概览为核心。

## 全局视觉规则

### 色彩

| 角色 | 颜色 | 用途 |
|------|------|------|
| Primary | `#0891B2` | 主操作、重点状态、链接 |
| Secondary | `#0F172A` | 标题、深色文字、层级强调 |
| Accent | `#22C55E` | 成功、已处理、正向状态 |
| Warning | `#F59E0B` | 待处理、提醒 |
| Danger | `#EF4444` | 错误、删除 |
| Background | `#F8FAFC` | 页面底色 |
| Surface | `#FFFFFF` | 卡片、弹窗 |
| Text | `#0F172A` | 正文 |
| Muted | `#64748B` | 辅助说明 |

### 字体

- **Heading Font:** Inter
- **Body Font:** Inter
- **字重:** 标题 600-700，正文 400-500
- **行高:** 正文不低于 1.5

### 间距与圆角

- 页面整体采用 24px / 32px 纵向节奏。
- 卡片圆角统一为 16px 或 20px。
- 按钮、标签、输入框保持统一的圆角节奏，不混用多个半径体系。

### 阴影

| Level | 用途 |
|-------|------|
| `--shadow-sm` | 轻量浮层 |
| `--shadow-md` | 卡片、下拉 |
| `--shadow-lg` | 弹窗、抽屉 |

## 组件规范

### 按钮

- 主按钮使用 Primary 色。
- 次按钮使用浅底深字或描边样式。
- 所有可点击元素必须有 `cursor: pointer`。
- Hover 过渡 150-300ms，反馈克制。

### 卡片

- 使用白底卡片承载数据块。
- 首选信息密度高、层级清晰的布局。
- 不要把页面 section 做成漂浮营销卡。

### 表格

- 默认使用紧凑表格密度，便于运营扫描。
- 表头固定、行高一致、悬浮高亮清晰。
- 筛选条件应尽量前置在表格上方。

### 表单

- 用于创建、编辑、配置的表单应保持简洁、可读。
- 复杂表单用分组、分区、步骤条或左右布局。

### 时间线 / 审计

- 审计、工单历史、通知历史优先使用时间线或事件流。
- 事件描述必须短，状态变化清晰。

## 交互规则

- 响应式必须兼容 375 / 768 / 1024 / 1440 四档。
- 鼠标和键盘都要可用，焦点态清晰。
- 禁止 emoji 作为图标。
- 图标仅使用 Lucide React 或 Ant Design Icons。
- 避免过重渐变和装饰性背景，信息优先。

## 反模式

- Hero/营销区块
- testimonial / logo wall / contact sales 叙事
- 过度装饰的渐变背景
- 大面积暗色主题
- 不必要的浮动圆角卡片堆叠
- 没有筛选入口就展示重表格

## 页面构建顺序建议

1. 顶部摘要指标
2. 筛选与搜索区
3. 主体表格或列表
4. 操作面板 / 详情抽屉 / 弹窗
5. 审计或时间线补充信息

## 可复用布局语义

- 统计卡片
- 操作工具栏
- 数据表格
- 详情抽屉
- 时间线
- 空状态
