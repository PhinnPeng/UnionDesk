# Design System Master File

> **逻辑**：构建任意页面前，先查看 `design-system/pages/[page-name].md`。
> 如果页面覆盖文件存在，则以页面覆盖为准；否则严格遵循本文件。

---

**Project:** UnionDesk Customer
**Category:** 客户服务门户 / 工单支持台

## 设计定位

- 面向终端客户，强调清晰、友好、专业、可操作。
- 目标是让客户快速提交工单、查看进度、处理通知、完成配置。
- 不使用 FAQ / 文档站 / 营销落地页 的结构。
- 不使用事件、会议、内容站、知识库首页等叙事方式。

## 全局视觉规则

### 色彩

| 角色 | 颜色 | 用途 |
|------|------|------|
| Primary | `#0891B2` | 主操作、链接、重点状态 |
| Secondary | `#0F172A` | 标题、深色文字 |
| Accent | `#22C55E` | 成功、已解决、正向反馈 |
| Warning | `#F59E0B` | 待处理、提醒 |
| Danger | `#EF4444` | 错误、删除 |
| Background | `#F8FAFC` | 页面底色 |
| Surface | `#FFFFFF` | 卡片、弹窗 |
| Text | `#162338` | 正文 |
| Muted | `#64748B` | 辅助说明 |

### 字体

- **Heading Font:** Inter
- **Body Font:** 思源黑体 / Noto Sans SC
- **字重:** 标题 600-700，正文 400-500
- **行高:** 正文不低于 1.5

### 间距与圆角

- 页面保持轻松、通透，但不松散。
- 卡片圆角统一为 16px 或 20px。
- 表单、按钮、标签、输入框保持统一的圆角节奏。

### 阴影

| Level | 用途 |
|-------|------|
| `--shadow-sm` | 轻量浮层 |
| `--shadow-md` | 卡片、下拉 |
| `--shadow-lg` | 弹窗、抽屉 |

## 组件规范

### 按钮

- 主按钮使用 Primary 色。
- 次按钮保持浅底深字或描边。
- 所有可点击元素必须有 `cursor: pointer`。
- Hover 过渡 150-300ms，反馈要轻。

### 卡片

- 使用白底卡片承载信息。
- 首页和详情页都要保持干净、友好、可读。
- 不要把页面 section 做成营销卡片。

### 表单

- 登录、注册、提单、设置表单要简洁、明显、可补救。
- 复杂配置使用分组和适度留白。

### 列表 / 时间线

- 工单、通知、回复、历史记录优先使用列表或对话流。
- 事件状态要可扫读，不要堆叠太多修饰。

## 交互规则

- 响应式必须兼容 375 / 768 / 1024 / 1440 四档。
- 鼠标和键盘都要可用，焦点态清晰。
- 禁止 emoji 作为图标。
- 图标仅使用 Lucide React 或 Ant Design Icons。
- 避免大面积暗色主题，页面以浅色、干净、可信为主。

## 反模式

- FAQ / Documentation Landing
- Hero / 证言 / CTA 叙事
- 过度装饰的渐变背景
- 会议 / 活动 / 内容站语义
- 深色科技风首页
- 过高的视觉噪音

## 页面构建顺序建议

1. 顶部信息区或状态摘要
2. 主要操作区
3. 列表、对话流或卡片网格
4. 通知 / 附件 / 历史补充信息
5. 设置与帮助入口

## 可复用布局语义

- 统计卡片
- 快速提单区
- 工单列表
- 通知列表
- 会话气泡
- 设置面板
- 空状态
