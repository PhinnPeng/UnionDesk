# Ticket Detail Override

## Page Intent

- Customer ticket detail page focused on conversation, status, attachments, and reply actions.
- The visual model is a support thread, not a social feed or event timeline.

## Required Structure

- Ticket info card:
  - 编号
  - 类型
  - 优先级
  - 状态
  - SLA 倒计时
- Conversation area:
  - 客户消息
  - 客服消息
  - 系统消息
- Reply composer:
  - text input
  - attachment entry
  - submit action

## Interaction Rules

- Primary actions: 回复、撤回、关闭
- Secondary actions: 附件、查看详情、返回列表
- Use a clean thread layout with clear sender separation.

## Visual Rules

- Keep the page calm and readable.
- Avoid wedding / event / conference visual language.
- Keep status chips and action buttons compact and aligned.
