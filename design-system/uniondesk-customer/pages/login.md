# Login Override

## Page Intent

- Customer authentication entry for sign in, registration, and password reset.
- This is a working support entry point, not a landing page.

## Required Structure

- Centered auth card or split layout with the form as the primary focus.
- Keep login / register / reset flow within one coherent surface.
- Slider captcha remains visible when enabled.

## Visual Rules

- Use support-portal colors and restrained branding.
- Keep copy short and direct.
- Avoid promotional hero imagery, testimonials, or event-style layouts.

## Interaction Rules

- Inputs, tabs, and buttons must stay clear and accessible.
- Loading and error states should feel calm and legible.
