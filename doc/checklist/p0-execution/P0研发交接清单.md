# P0 研发交接清单

> 来源：`doc/产品需求文档 v1.0.md`、`doc/P0接口契约表.md`、`doc/P0验收用例表.md`、`doc/checklist/p0-execution/P0开发方案.md`。
> 执行 Gate 表见 [P0验收Gate表.md](./P0验收Gate表.md)。
> 范围：仅覆盖 P0 必交付项；`TC-019` 为 P1 参考，不阻塞 P0。
> 目标：将 PRD 拆成可分配给后端、前端、数据库、测试/交付的研发任务。

## 0. 交接原则

1. P0 必须通过 53 条 P0 验收用例；`TC-019` 仅用于 P1 参考验证。
2. 所有接口以 `doc/P0接口契约表.md` 为准；接口字段变更必须同步 PRD、验收用例和前端 shared types。
3. `TicketController` 等 demo 链路只能用于临时验证；进入工单 Sprint 后必须替换为持久化领域服务。
4. MinIO 和 SMTP 在 Sprint 0 冻结决策；不可用时按 PRD 降级，不阻塞 P0。
5. 每项任务完成后必须更新本清单状态、补充验证结果和未关闭风险。

## 6. 交接顺序

1. 先做 `P0-G0-*`：稳定启动链路、Flyway、依赖决策。
2. 并行做 `P0-BE-AUTH-*`、`P0-BE-IAM-*`、`P0-DB-01~05`、`P0-FE-BASE-*`。
3. 身份与域闭环后，集中做 `P0-BE-TICKET-*`、`P0-DB-06~09`、`CustomerWeb` 和 `AdminWeb` 工单页面。
4. 工单闭环后补齐 SLA、通知、安全、审计、交付任务。
5. 最后按 `P0-QA-*` 跑完整验收，不通过项回填到对应任务。
