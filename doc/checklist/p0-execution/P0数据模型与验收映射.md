## 2. 表结构 ↔ P0 验收用例（TC）覆盖关系

| 表或对象 | 主要支撑能力 | 典型 TC |
|----------|----------------|---------|
| `identity_subject` / `staff_account` / `customer_account` / `identity_binding` | 统一身份、双账号、第三方绑定预留 | TC-004、TC-010、P1 SSO |
| `business_domain` + `registration_policy` / `visibility_policy_codes` / `deleted_at` | 域可见策略、注册策略、软删 | TC-001、TC-002、TC-003 |
| `domain_member` / `domain_role` / `domain_member_role` / `permission_item` / `domain_role_permission` | 域成员与域角色（目标态，与 legacy `role` 并存） | TC-008、TC-009、TC-042、TC-043 |
| `platform_role` / `staff_account_platform_role` | 平台角色绑定 | TC-012、TC-053 |
| `domain_customer` / `invitation_code` | 客户入域、邀请码 | TC-003 |
| `domain_config` / `system_config` / `domain_template` | 域配置、平台配置、域模板 | TC-045、复制域 |
| `ticket` + 扩展列（`version`、`sla_*`、`customer_account_id`、`assignee_staff_account_id` 等） | 乐观锁、SLA 计时、新身份外键（可空过渡） | TC-013~TC-017、TC-041、TC-048 |
| `ticket_type.status_flow_config` | 类型级状态机配置 | TC-040、状态机表 |
| `ticket_history` | 工单操作流（与 `ticket_event_log` 并存，迁移期双写可选） | TC-047、TC-016 |
| `ticket_relation` | 合并/跟进关联 | TC-038 |
| `ticket_template` / `quick_reply_template` | 工单模板、快捷回复 | TC-039、TC-040、TC-046 |
| `ticket_priority_level` | 域级优先级 | TC-041 |
| `dynamic_field_config` | 动态字段（PRD `dynamic_field_config`） | TC-040 |
| `sla_calendar` / `sla_rule` | 工作日历与 SLA 规则 | TC-018 |
| `file_attachment` / `attachment_ref` / `attachment_policy` | 统一附件与策略 | TC-035、TC-054 |
| `notification_template` / `notification_log` / `inbox_message` | 模板、发送轨迹、站内信 | TC-022、TC-023 |
| `audit_log` | PRD 审计（与 `operation_log` 并存） | TC-020、TC-049 |
| `login_log` | PRD 登录日志（与 `auth_login_log` 并存） | TC-021 |
| `blocked_word` | 关键字屏蔽 | TC-025 |
| `access_ip_policy` | IP 白名单 | TC-026 |
| `staff_presence` | 客服在线态 | 咨询/队列相关 |
| `customer_cancel_request` | 客户注销申请 | PRD 账户注销 |

**未在本脚本删除的 legacy 表**：`user_account`、`customer_business_domain_access`、`ticket_event_log`、`operation_log`、`auth_login_log`、`iam_*` 等仍供当前代码路径使用；应用层按《数据模型迁移策略.md》分阶段切换读写。

## 3. 幂等性与重复启动

- **应用重复启动**：Flyway 对已执行版本不重复执行 SQL，校验通过即可。
- **脚本内幂等**：新表 `CREATE IF NOT EXISTS`；`business_domain` / `ticket` / `ticket_type` / `ticket_reply` / `consultation_session` 的列与索引、外键通过 `information_schema` 条件追加，便于在测试库手工重放同一文件。
- **种子**：`platform_role`、`ticket_priority_level`、`attachment_policy` 使用 `INSERT … ON DUPLICATE KEY UPDATE`。

## 4. 索引与回滚

- **索引**：工单侧新增 `idx_ticket_assignee_staff`；审计/登录/站内信/通知按查询维度建二级索引（见 DDL）。
- **回滚**：无自动 down 迁移；生产回滚依赖备份恢复。修改已 checksum 的历史脚本（如 `V202604200001`）后，已部署环境需 DBA 执行 `flyway repair` 或清理 `flyway_schema_history` 对应行后重装（仅限非生产）。
