# 平台用户管理页面优化清单

## 目标

优化 `platform/user` 页面，在平台用户管理块内补齐部门树、搜索块和用户操作能力，并同步按钮级权限到菜单管理可识别的权限码。

## 计划

1. 梳理用户页数据结构和权限码 -> 验证现有用户、组织、角色接口与测试覆盖点
2. 增加用户管理工具函数与权限清单 -> 验证搜索、行转换、密码生成单测
3. 重构 `platform/user` 管理块 -> 验证部门树、搜索块、新增、编辑、离职、批量离职、更多操作渲染与交互
4. 调整用户弹窗为新增/编辑复用 -> 验证新增和编辑提交 payload
5. 同步菜单管理权限码 -> 验证权限码标签和前端类型检查

## 任务

- [x] 在用户管理块左侧添加部门树
- [x] 在用户管理块添加关键字搜索块，支持用户名、账号、手机号、邮箱模糊搜索
- [x] 支持新增、编辑、离职、批量离职和更多/重置密码操作
- [x] 补齐按钮级权限码与菜单管理标签
- [x] 补齐 BDD/单测并执行前端校验

## 验收

- 具备权限的用户可以看到搜索、新增、批量离职、编辑、离职、更多/重置密码入口
- 关键字搜索覆盖用户名、账号、手机号、邮箱
- 新增、编辑、离职、批量离职、重置密码操作会同步当前用户表格数据
- 重置密码生成 16 位密码，包含数字、小写字母、大写字母、常见字符
- 菜单管理可识别本次新增/使用的按钮级权限码

## 约束

- 仅修改 `platform/user` 页面、其测试、相关权限标签/权限码清单与必要 API 支撑
- 不改登录、路由和无关业务页面
- 当前用户数据没有部门归属字段，部门树仅展示组织结构，不做伪造筛选
