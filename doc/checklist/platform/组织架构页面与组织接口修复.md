# 组织架构页面与组织接口修复清单

## 目标

修复 `GET /api/v1/iam/organizations` 的 500 问题，并将 `platform/dept` 调整为与 `system/menu` 一致的管理页风格，支持搜索块、Modal 表单块、Table 数据块，以及一键展开/缩回。

## 任务

- [x] 修复组织接口 500 的核心映射问题，恢复组织列表接口可用
- [x] 将 `platform/dept` 重构为 Table 数据块 + Modal 表单块
- [x] 支持部门名称/编码关键字搜索与创建时间搜索
- [x] 支持节点新增、编辑、删除，以及全量查看平台部门
- [x] 支持 Table 一键展开与一键缩回
- [x] 补齐 BDD/单测并完成浏览器验证
- [x] 收口新增/编辑部门弹窗：自动生成部门编码、调整字段顺序、移除“无上级部门”可点击项

## 验收

- `GET /api/v1/iam/organizations` 可正常返回组织数据
- `platform/dept` 采用搜索块 + Table 数据块 + Modal 表单块
- 搜索支持名称、编码和创建时间
- 节点新增、编辑、删除逻辑可用
- Table 支持一键展开与一键缩回
- 新增部门弹窗自动生成 `dept-` 前缀 8 位短码，编辑弹窗保留原编码
- 上级部门为空时只提交 `null`，不把“无上级部门”写入数据
- 相关单测与前端类型检查通过

## 约束

- 优先复用现有模块化结构，避免不必要的抽象
- 只修改组织架构和组织接口相关的必要代码
- 涉及后端时必须确保服务重新启动并可访问
