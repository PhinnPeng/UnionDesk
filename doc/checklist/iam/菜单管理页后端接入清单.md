# 菜单管理页后端接入清单

## 背景
菜单管理页 `/platform/menu` 需要从旧 mock 切换到真实后端菜单树接口，并修复按钮权限与表单字段对齐问题。

## 计划
1. 修复按钮权限判断
   - 在后端驱动路由场景下，`hasAccessByCodes()` 不再因为缺少 `handle.permissions` 直接禁用按钮。
   - 验证：`新增` 按钮恢复可点击。
   - 状态：已完成

2. 接入真实后端菜单 API
   - 将菜单列表、创建、更新、删除接口切换到 `/api/v1/iam/menus/*`。
   - 菜单页直接消费后端树结构，不再走旧 mock `/api/menu-list`。
   - 验证：菜单表格有数据，控制台无 404。
   - 状态：已完成

3. 对齐菜单表单与表格字段
   - 确认表格列和表单字段名与后端 `MenuTreeNode` / 菜单请求体一致。
   - 验证：新增、编辑弹窗可正常回填和提交。
   - 状态：已完成

## 检查点
- `pnpm --dir UnionDeskWeb/apps/UnionDeskAdminWeb typecheck` 通过
- `pnpm --dir UnionDeskWeb/apps/UnionDeskAdminWeb test --run` 通过
- 浏览器打开 `/platform/menu` 后，菜单表格正常展示，`新增` 按钮可点击，新增/编辑弹窗可正常操作
