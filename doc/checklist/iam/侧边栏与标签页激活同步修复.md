# 侧边栏与标签页激活同步修复清单

## 背景
后端路由的 `id` 已经改成了 `backend:*` 形式，`LayoutMenu` 仍在用 `useMatches()` 的 `id` 计算菜单选中态，导致当前页面虽然已经切换成功，但侧边栏无法高亮当前菜单。

## 计划
1. [x] 修正 `src/layout/layout-menu/index.tsx` 的选中态来源
   - 验证：`/platform/home`、`/platform/role` 等页面的侧边栏高亮能正确跟随当前路由
2. [x] 补充回归测试，覆盖 backend route id 场景
   - 验证：菜单选中态使用 `pathname`，不会再被 `backend:*` 的 `id` 覆盖
3. [x] 浏览器回归验证侧边栏与标签页联动
   - 验证：点击侧边栏菜单后，页面内容、标签页和侧边栏激活状态保持一致；点击标签页后，侧边栏同步切换
4. [x] 去掉激活 tab 关闭按钮的左侧间距
   - 验证：激活状态的可关闭标签页按钮不再出现 `margin-left: var(--ant-margin-xs)` 的偏移

## 状态
- [x] 已完成

## 结果
- `LayoutMenu` 选中态改为优先读取 `pathname`，避免 backend 路由 `id` 干扰
- 已补充 `getSelectedMenuKeys` 单元测试
- 已在浏览器中验证 `/platform/home` 与 `/platform/role` 的侧边栏和标签页联动正常
- 已去掉激活 tab 关闭按钮的左侧间距
